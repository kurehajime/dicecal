# 実装タスク

`docs/仕様.md` をもとに、サイコロカレンダーの実装タスクを以下のように分解する。

## 1. 開発基盤を整える

* `three`、`@react-three/fiber`、`@react-three/drei`、`@react-three/rapier` を追加する
* `Canvas` を表示できる最小構成を作る
* 3D 関連コンポーネントのディレクトリ構成を決める
* Rapier を初期実装から組み込み、物理層を前提にした構成にする

成果物:
3D シーンが表示される初期画面

## 2. 仕様を型に落とす

* 4 つのサイコロの役割を型で定義する
* 各面の値を定数として定義する
* 初期姿勢、配置、確定済み姿勢を表すデータ構造を作る

成果物:
`DiceKind`、面ラベル定義、配置定義、回転状態モデル

## 3. シーン全体を組む

* 台座、カメラ、照明を実装する
* 4 つのサイコロを斜め配置で並べる
* 仕様どおりの初期レイアウトを再現する

成果物:
見た目として成立した 3D のカレンダー配置

## 4. サイコロの見た目を作る

* 各サイコロに 6 面のテキストを描画する
* 月、曜日、日付 1、日付 2 の各サイコロを共通コンポーネントで表現する
* 面ラベルが視認できるサイズと質感を調整する

成果物:
面ラベルが正しく見える 4 種類のサイコロ

## 5. 選択操作を実装する

* 任意のサイコロをクリックして選択できるようにする
* 選択されたサイコロが持ち上がる動きを実装する
* 選択されたサイコロが画面正面を向くようにする

成果物:
クリックで選択状態へ遷移する UI

## 6. 選択中 UI を出す

* 選択されたサイコロの周囲に回転用の矢印を表示する
* 確定用のチェックマークを表示する
* 操作 UI は 3D 空間内ではなく画面固定オーバーレイとして表示する
* 非選択時には UI を非表示にする

成果物:
選択中だけ操作 UI が見える状態

## 7. 回転ロジックを実装する

* 上下左右の回転処理を実装する
* 追加の左右回転は、正面を維持したままの Z 軸回転として実装する
* 1 回の操作で 90 度回転するようにする
* 面の向きが破綻しないように状態を更新する

成果物:
UI 操作で正しい面が前面に来る回転処理

## 8. 確定処理を実装する

* チェック操作でサイコロを元の位置に戻す
* 確定した向きを保持する
* 再選択時に前回確定した向きを再現する

成果物:
確定後の状態が維持される挙動

## 9. アプリ状態を整理する

* `未選択`、`選択中`、`回転中`、`確定済み` の状態を整理する
* 複数サイコロの状態を一元管理する
* UI 状態と 3D 表示状態の同期を取る

成果物:
状態管理の破綻がない構成

## 10. アニメーションと操作感を調整する

* 持ち上がり、正面向き、回転、復帰の補間アニメーションを入れる
* クリック判定を調整する
* カメラ距離や見やすさを最適化する

成果物:
自然な操作感のある UI

## 11. 永続化を入れる

* 各サイコロの向きを保存する
* ページ再読込時に保存済み状態を復元する
* 保存先は `localStorage` を第一候補とする

成果物:
リロード後も状態が再現される

## 12. 検証と仕上げ

* 全サイコロの面が仕様どおりか確認する
* 回転操作で矛盾が出ないか確認する
* 画面サイズが変わっても致命的に崩れないか確認する
* 最低限の lint と build を通す

成果物:
動作確認済みの完成版

## 推奨する実装順

1. 開発基盤
2. 型定義
3. シーン構築
4. サイコロ表示
5. 選択
6. 回転
7. 確定
8. 保存
9. 調整と検証

## 決定事項

* 回転 UI は画面固定オーバーレイとして実装する
* 「左右に回転する矢印」は Z 軸の時計回り・反時計回りとして実装する
* Rapier は初期実装から導入する
