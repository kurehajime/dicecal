# バックエンド

## 方針
- Firebase Realtime Database を使う
- クライアントから直接 read / write する
- コードはこのリポジトリで管理する
- 専用のサーバーは置かない
- データは公開読み書きとする

## 認証
- ログインは設けない
- Firebase Authentication は使わない

## 保護
- App Check を併用する
- 更新対象は最小限の 1 ノードに寄せる

## データ検証
- Realtime Database Rules の `.validate` で型と構造を検証する
- クライアント側でも同じスキーマで検証する
- 削除禁止は `.write` 側でも制御する

## 保存対象
- サイコロの並び順
- 各サイコロの向き
- 一時 UI 状態は保存しない

## 注意
- 公開書き込みなので荒らし耐性は弱い
- 壊れても復旧できる前提で運用する
- App Check は abuse 軽減には有効だが認証の代わりにはならない
