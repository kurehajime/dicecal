# バックエンド

## 方針
- Firebase Realtime Database を使う
- クライアントから直接 read / write する
- コードはこのリポジトリで管理する
- 専用のサーバーは置かない

## 認証
- Firebase Anonymous Auth を使う
- 通常のログイン UI は設けない
- 見た目はログイン不要のまま運用する
- Realtime Database Rules で `auth.uid` を使う

## 保護
- App Check を併用する
- 匿名公開書き込みにはしない

## データ検証
- Realtime Database Rules の `.validate` で型と構造を検証する
- クライアント側でも同じスキーマで検証する
- 削除禁止は `.write` 側でも制御する

## 保存対象
- サイコロの並び順
- 各サイコロの向き
- 一時 UI 状態は保存しない

## 注意
- Anonymous Auth は UI なしで使えるが、Firebase 側では認証ありとして扱う
- App Check は abuse 軽減には有効だが認証の代わりにはならない
